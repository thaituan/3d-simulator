# public/models/ — 3Dモデル管理ディレクトリ

## 概要

このディレクトリには、ブラウザで表示するための3Dモデルファイル（GLB形式）を配置しています。

将来的にはS3等のクラウドストレージでモデルを管理する想定ですが、
本リポジトリは検証用のため、`public/models/` 内で直接管理しています。

## ディレクトリ構造

```
public/models/
├── README.md                      ← 本ファイル
├── catalog.json                   ← 全モデルのメタデータ(商品名など本来はAPIで取得する情報)
├── {productCode}.glb              ← 各モデルファイル
├── ...
└── textures/
    └── MS02/                      ← MS02ソファのテクスチャスワップ検証用
        ├── fabriclightgray.jpg
        ├── fabricgray.jpg
        └── ...
```

## 配置ルール

### GLBファイル

- **フラット構造（1階層）** で全モデルを配置する
- ファイル名は **商品コード.glb**（例: `3000591.glb`, `2110300007864-2110300016744.glb`）
- シリーズ別のサブフォルダは作らない（分類は `catalog.json` のメタデータで行う）
- MS02ソファは2つの商品コードをハイフン繋ぎにしたファイル名を使用する

### catalog.json

- 全モデルのメタデータを持つJSONファイル
- `npm run convert-models` で自動生成される
- アプリ側から `fetch('/models/catalog.json')` で取得して使用する
- ニトリネットAPIの代替としてfetchを行う仕様にした

### textures/

- テクスチャスワップ（ランタイムでの色切替）検証用のテクスチャ画像
- 現時点ではMS02ソファのファブリック色テクスチャのみ
- 他シリーズのテクスチャスワップを行う場合は `textures/{seriesId}/` に追加する

## GLB形式を採用した理由

元データはデジ販の管理フォルダにあるFBX形式のファイルです。
以下の理由からGLB形式に変換して使用しています。

1. **テクスチャ埋め込み** — テクスチャを含めて1ファイルに収まるため管理がシンプル
2. **R3F互換** — React Three Fiber の `useGLTF`（drei）でそのまま読み込める
3. **Web標準** — glTF/GLBはWeb3Dの標準フォーマットとして設計されている
4. **S3親和性** — 1ファイルのアップロード・配信で済む
5. **RITTAI** - そもそもFBXとして納品されてはいるが、RITTAIで表示する際はGLB形式を使っている

## 変換方法

```bash
npm run convert-models
```

このコマンドで以下が行われます:

1. `temp/3dModel/` 内のFBXファイルを走査
2. `fbx2gltf`（Meta/Facebook製）でGLB形式に変換
3. 変換済みGLBファイルを本ディレクトリに出力
4. `catalog.json` を自動生成
5. MS02のファブリックテクスチャを `textures/MS02/` にコピー

## 元ファイルの所在

- 変換前のFBXファイルは `temp/3dModel/` に格納されています
- `temp/` はGit管理外（`.gitignore` で除外）のため、リポジトリには含まれません
- FBXファイルはデジ販の管理フォルダから取得したものです
